# plnv-canvas-demo

Демонстрация `@plnv/canvas` на Angular 21.

## Что здесь происходит

Один родительский `<canvas plnvCanvas2d>` задаёт контекст 2D и общую отрисовку. Внутри него в шаблоне лежат **дочерние узлы**-слои (`<canvas app…Layer>`): каждый слой — директива, наследующая `PlnvCanvasElement`, со своей функцией `draw`. Сервис библиотеки собирает очередь слоёв и каждый кадр очищает канвас и вызывает слои по порядку — так демонстрируется **композиция нескольких «плоскостей» в одном буфере** без вложенных реальных canvas в DOM смысле стека.

## Что демонстрирует интерфейс

| Элемент | Смысл |
|--------|--------|
| **Горы** (`MountainLayer`) | Нижний слой сцены: небо и три ряда силуэтов; новый ландшафт по «Перерисовать»; можно скрыть кнопкой. |
| **Солнце / луна** (`SunMoonLayer`) | Поверх гор: пунктир орбиты в общем контексте; диск и ореол — offscreen → `drawImage`. Позиция по времени суток; в тёмной теме фаза луны от **дня месяца** (13-е — полнолуние). |
| **Снег** (`SnowLayer`) | Верхний слой: `needRedraw`, снежинки, отталкивание от курсора. |
| **Тема** | По умолчанию как в ОС (`prefers-color-scheme`); кнопка переключает цикл: системная → светлая → тёмная → системная. `data-theme` на `<html>`. |

Импорт `@plnv/canvas` резолвится через **`paths`** в `tsconfig.json` / `tsconfig.app.json` на исходники библиотеки (`../src/public-api.ts`). Сборка **`dist/`** для работы примера **не нужна** — достаточно `npm install` в `example/` и `ng serve`.

```bash
cd example && npm install && npx ng serve
```

Запуск:

```bash
cd example && npm run serve
# или npm start
```

Из **корня** репозитория:

```bash
npm run example          # сначала npm run build (библиотека), затем ng serve
npm run example:serve    # только ng serve в example/ (dist для примера не обязателен)
```

Откройте URL из вывода CLI (обычно http://localhost:4200/).
